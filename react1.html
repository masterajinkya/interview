2. What is difference between virtual dom and shallow dom , dom in React js
Ans:

1.Dom(Document Object Model) =>

  • The DOM is the actual representation of the structure of an HTML document in a web browser.
  • It's a tree-like structure where each HTML element (e.g., div, span, p) is represented as a node in the tree.
  • Manipulating the DOM directly in JavaScript can be slow and inefficient. Beacuse it update immediately the real dom

2.Virtual dom(Virtual Document Object Model) =>

  • The Virtual DOM is a Copy of actual DOM.
  • It's a lightweight, in-memory representation of the actual DOM.
  • When you make changes to a component's state or props, React doesn't immediately update the real DOM. Instead, it creates a new Virtual DOM tree that represents what the updated UI should look like.
  • React then performs a process called "reconciliation,"

3.Shallow Dom(Shallow Document Object Model) =>

  • Shallow DOM is not a widely used React.
  • In web components, the Shadow DOM allowing components to have their own isolated DOM tree that doesn't interfere with the rest of the page's DOM
  • React doesn't use the Shadow DOM by default. Instead, it relies on the Virtual DOM to manage updates efficiently.

9. Explain data flow in Redux
Ans:

Redux follows the unidirectional data flow. It means that your application data will follow in one-way binding data flow.

  1. Action Dispatch:

    • The data flow begins when an action is dispatched. Actions are plain JavaScript objects that describe an event or an intention to change the state.
    • Each action must have a type property that defines the type of action being performed. Additional data, often referred to as the "payload,".

  2. Reducers:

    • Reducers are pure functions responsible for handling the state updates based on the dispatched action.
    • When an action is dispatched, all registered reducers are called.
    • Each reducer takes the current state and the action as arguments and returns a new state object.
    • Reducers must not modify the existing state but create a new state object that reflects the changes.
    • Reducers typically use a switch statement to determine how to update the state based on the action's type.

  3. Store Update:

    • After all reducers have been executed, the new state returned by each reducer is combined into a single state object.
    • This new state object becomes the updated state of the Redux store.

  4. UI Component Updates:

    • React components that are connected to the Redux store through the connect function (provided by the React-Redux library) receive updates when the state in the Redux store changes.
    • These components can specify which parts of the state they are interested in using selectors. Selectors are functions that extract specific data from the application state.

  5. Render UI:

    • When a component receives updated state from the store, it re-renders with the new data.
    • Components are re-rendered efficiently, as React uses a virtual DOM to determine the minimal changes needed to update the actual DOM.

  6. User Interaction:

    • User interactions with the UI, such as clicking buttons or entering text, can trigger new actions to be dispatched.
    • This starts the data flow cycle again, with new actions being dispatched, processed by reducers, and resulting in updated state.

12. Difference between class component and function component
Ans:

Class Components:

  1. Class Syntax: Class components are defined using JavaScript ES6 class syntax. They must extend the React.Component base class.

  2. State Management: Class components can manage their own internal state using the this.state object. State changes trigger component re-renders.

  3. Lifecycle Methods: Class components have access to lifecycle methods such as componentDidMount, componentDidUpdate, and componentWillUnmount. These methods are used for side effects, data fetching, and cleanup operations.

  4. Complex Logic: They are suitable for components with complex logic, state management, or lifecycle requirements.

  5. Ref Usage: Refs can be used to access and interact with DOM elements or child components.

Function Components:

  1. Function Syntax: Function components are defined as JavaScript functions that return JSX. They are simpler and more concise than class components.

  2. State Management: Function components can use state management via the useState hook, introduced in React 16.8. Hooks allow functional components to manage state and other React features previously available only in class components.

  3. Lifecycle with Hooks: Function components can replicate the behavior of class component lifecycle methods using hooks like useEffect for side effects and data fetching.

  4. Functional Paradigm: They are ideal for components with simpler logic and for adhering to the functional programming paradigm.

  5. Ref Usage: Function components can also use the useRef hook to access DOM elements or child components, similar to class component refs.

13. How can we implement componentWillUnmount in function component
Ans:

In function components in React, you can't use componentWillUnmount directly because it's part of the class component lifecycle. Instead, you can achieve the same cleanup behavior using the useEffect hook. Here's how you can implement the equivalent of componentWillUnmount in a function component:

When Does useEffect Run:

  • No Dependency Array: If you pass an empty dependency array ([]) as the second argument, the effect runs once after the initial render. It's equivalent to the behavior of componentDidMount in class components.

  • With Dependencies: If you provide an array of dependencies, the effect will run whenever any of the dependencies change. This allows you to control when the effect runs based on specific state or prop changes. It's similar to componentDidUpdate in class components.

  • No Second Argument: If you omit the second argument, the effect runs after every render, which is often not what you want. It's equivalent to not using shouldComponentUpdate in class components.

Cleanup:

  • You can return a cleanup function from the effect. This function is called when the component is unmounted or when the dependencies change before the next effect run. It allows you to clean up any resources or subscriptions to prevent memory leaks.

  • Usage:

16. What is difference between export default and export in React js
Ans:

  1. export default:

    • When you use export default, you are exporting a single value or object as the default export for your module.
    • There can be only one default export per module.
    • You can give the default export any name you want when importing it in another module.
    • Default exports are commonly used for exporting the main or primary functionality of a module.

  2. export:

    • When you use export, you are exporting one or more named exports. You can have multiple named exports in a single module.
    • Named exports require you to specify the name when importing them in another module.
    • Named exports are useful for exporting multiple values, functions, or objects from a module.
