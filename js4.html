77.What is call by value and call by reference in JavaScript?
Ans - In JavaScript, when passing arguments to functions, it's important to understand the concepts of "call by value" and "call by reference," although these terms can sometimes be misleading due to the way JavaScript handles data types.

Call by Value -
In languages that strictly follow call by value, a copy of the actual value of the variable is passed to the function. This means that changes made to the parameter within the function do not affect the original value outside the function. JavaScript's primitive data types (such as numbers, strings, booleans) behave like call by value

Call by Reference -
In languages that strictly follow call by reference, a reference or memory address of the variable is passed to the function. This means that changes made to the parameter within the function affect the original value outside the function. JavaScript's objects (including arrays and functions) behave like call by reference.

81.Difference between JavaScript and jQuery?
Ans -
JavaScript:-

1 Language Nature:
JavaScript is a programming language. It is a versatile and widely-used scripting language that can be used for a variety of purposes, including web development, server-side programming (Node.js), and more.
2 Execution Environment:
JavaScript can run in various environments, including web browsers, servers (Node.js), mobile apps, and more.
3 DOM Manipulation:
JavaScript allows direct manipulation of the Document Object Model (DOM), which represents the structure of a web page. It provides native methods to interact with HTML elements, attributes, and events.
4 Learning Curve:
Learning JavaScript requires understanding its core concepts, including variables, data types, functions, control structures, and object-oriented programming (OOP) principles.
5 Browser Compatibility:
JavaScript code can sometimes require cross-browser testing and adjustments to ensure compatibility across different browsers.

jQuery:-

Library:
jQuery is a JavaScript library. It is not a programming language but rather a set of pre-written JavaScript functions and methods packaged in a convenient API.
Simplification:
jQuery simplifies common tasks in web development, such as DOM manipulation, event handling, animations, and AJAX requests. It provides a concise syntax for performing these tasks, often requiring less code compared to plain JavaScript.
DOM Manipulation:
jQuery abstracts and simplifies DOM manipulation, providing a consistent interface across different browsers. This can help mitigate some browser compatibility issues.
Learning Curve:
Learning jQuery is often faster than learning pure JavaScript due to its simplified API. However, it's important to have a foundational understanding of JavaScript concepts to use jQuery effectively.
Browser Compatibility:
jQuery handles many browser compatibility issues internally, making it easier to write code that works consistently across browsers.
