2 What is prototype chain in js?

Ans:

  • The prototype chain is formed by linking objects to their prototypes.
  • The prototype chain is a mechanism that allows objects to inherit properties and methods from other objects
  • Every object can have exactly one prototype object. That prototype object can also have a prototype object, and so on
  • Each object has a private property which holds a link to another object called its prototype. That prototype object has a prototype of its own, and so on until an object is reached with null as its prototype. By definition, null has no prototype, and acts as the final link in this prototype chain.

JavaScript has several built-in data types, which can be categorized into two main categories:

5 Difference between == and === in js?

Ans:

In JavaScript, both == (loose equality) and === (strict equality) are used for comparison, but they have different behaviors and comparison rules:

  1. == (Equality Operator):

    • The == operator checks for equality of values while performing type coercion if the operands have different data types.
    • Type coercion means that JavaScript may convert one or both operands to a common type before making the comparison.
    • For example, 5 == "5" would return true because JavaScript converts the string "5" to the number 5 before making the comparison.

  2. === (Strict Equality Operator):

    • The === operator checks for equality of values without performing type coercion.
    • It requires both the value and the data type to be the same for the comparison to return true.
    • For example, 5 === "5" would return false because the types are different (number vs. string).

                console.log(5 == "5");    // true (type coercion)
                console.log(5 === "5");   // false (strict comparison)

                console.log(true == 1);   // true (type coercion)
                console.log(true === 1);  // false (strict comparison)

                console.log(null == undefined);   // true (type coercion)
                console.log(null === undefined);  // false (strict comparison)

10 What is closures? && Q.82 Advantages of closure?

Ans:

A closure is a feature of JavaScript that allows inner functions to access the outer scope of a function. 

function init() {
                var name = "Mozilla"; // name is a local variable created by init
                function displayName() {
                  // displayName() is the inner function, that forms the closure
                  console.log(name); // use variable declared in the parent function
                }
                displayName();
              }
              init();

Advantages
  1. Encapsulation: Closures allow you to create private variables and functions that can only be accessed from within the closure. This can help you improve the structure and modularity of your code.
  2. State retention: Closures allow you to retain the state of a function even after it has finished running. This can be handy for creating function factories or other patterns that rely on maintaining state across multiple function calls.
  3. Currying: Closures can help you create curried functions, which are functions that can be called with a partial set of arguments and then return a new function that expects the remaining arguments. This can make your code more flexible and reusable, which is always a good thing!
  4. Memorization: Closures can be used to implement the memorization technique, which allows a function to remember and reuse its previous results. This can help you improve the performance of your code.
  5. Asynchronous programming: Closures can simplify asynchronous programming by allowing you to define and execute callback functions or async logic within the context of the asynchronous operation, rather than having to define them separately and pass them around as arguments.
  6. Event handling: Closures can be used to create event handlers that have access to variables and functions defined in their parent scope. This can be really helpful for implementing complex event-driven behavior in your code.
11 What is scope in js?

Ans:

Scope determines the accessibility of variables, objects, and functions from different parts of the code.

Scope determines the accessibility (visibility) of variables.

JavaScript has two main types of scope:

1. Global Scope: Variables and functions declared outside of any function or block (i.e., not enclosed within curly braces {}) have global scope. They are accessible from anywhere in your JavaScript code, including within functions and blocks.

const globalVar = 42;
function globalFunction() {
console.log(globalVar); // Accessible from inside the function
}
console.log(globalVar);
// Accessible from outside the function

Local Scope: Variables and functions declared inside a function or block have local scope. They are only accessible within the function or block in which they are declared. Local scope variables are often called "local variables."

function localScopeExample() {
                    const localVar = 10;
                    console.log(localVar); // Accessible within the function
                }

                localScopeExample();

                console.log(localVar); // ReferenceError: localVar is not defined

12 What is web storage?

Ans:

Web storage is a web browser feature that allows web applications to store data locally on a user's device. It provides a simple and efficient way to store key-value pairs.

1.localStorage: This is a type of web storage that allows you to store data with no expiration time. The data stored in localStorage remains available even after the browser is closed and reopened, making it suitable for long-term storage. Each key-value pair you store in localStorage is associated with a specific domain, and other web pages from the same domain can access this data. 


                
// Storing data in localStorage
                localStorage.setItem("username", "john_doe");

                // Retrieving data from localStorage
                const username = localStorage.getItem("username");

                // Removing data from localStorage
                localStorage.removeItem("username");

2.sessionStorage: This is similar to localStorage, but it stores data for the duration of a page session. The data is available as long as the page is open and is cleared when the page is closed or refreshed. sessionStorage is useful for temporarily storing data that you need only for a single page load.


                    
// Storing data in sessionStorage
                    sessionStorage.setItem("theme", "dark");

                    // Retrieving data from sessionStorage
                    const theme = sessionStorage.getItem("theme");

                    // Removing data from sessionStorage
                    sessionStorage.removeItem("theme");

15 Difference between local storage and session storage?

Ans:

  1. localStorage:

    • localStorage is a type of web storage that allows data to be stored persistently in the web browser across sessions.
    • Data stored in localStorage has no expiration date and will remain available even after the browser is closed and reopened.
    • The data stored in localStorage is accessible from the same domain and protocol that set the data. It means data stored on one page of your website can be accessed by other pages within the same domain.
    • It is important to note that localStorage has a size limit, typically around 5-10 MB, depending on the browser.

  2. sessionStorage:

    • sessionStorage is another type of web storage that stores data for a single session. A session lasts as long as the browser tab or window remains open, and the data is available across multiple pages during that session.
    • Unlike localStorage, data stored in sessionStorage is cleared when the browser tab or window is closed. It is meant for temporary storage that is only relevant during a specific browsing session.
    • As with localStorage, data stored in sessionStorage is accessible from the same domain and protocol that set the data.
